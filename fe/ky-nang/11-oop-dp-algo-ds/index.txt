18:T2f98,<h1>OOP / Design Pattern / Algorithm / Data Structure</h1>
<p><strong>Định nghĩa.</strong> Bốn nền tảng khoa học máy tính dùng chung cho mọi dev:</p>
<ul>
<li><strong>OOP</strong>: tổ chức code quanh <em>đối tượng</em> với đóng gói, kế thừa, đa hình.</li>
<li><strong>Design Pattern</strong>: lời giải mẫu cho bài toán thiết kế lặp lại (Singleton, Factory, Observer...).</li>
<li><strong>Algorithm</strong>: cách giải + đánh giá chi phí (Big-O).</li>
<li><strong>Data Structure</strong>: cách tổ chức dữ liệu (mảng, Map, cây...) — chọn đúng thì nhanh &amp; gọn.</li>
</ul>
<p>Ở FE (TypeScript/Angular) chúng không hàn lâm: chọn đúng cấu trúc và pattern quyết định code dễ bảo trì và chạy nhanh không.</p>
<h2>▸ Entry — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu class/interface, đóng gói; pattern đơn giản; độ phức tạp cơ bản (O(n) vs O(n²)); cấu trúc thường dùng (array, <code>Map</code>).</p>
<p><strong>Ví dụ thực tế — chọn <code>Map</code> thay vì lặp lồng nhau.</strong></p>
<pre><code class="language-typescript"><span class="hljs-comment">// ❌ O(n*m): với mỗi đơn quét lại toàn bộ user</span>
orders.<span class="hljs-title function_">forEach</span>(<span class="hljs-function"><span class="hljs-params">o</span> =&gt;</span> o.<span class="hljs-property">userName</span> = users.<span class="hljs-title function_">find</span>(<span class="hljs-function"><span class="hljs-params">u</span> =&gt;</span> u.<span class="hljs-property">id</span> === o.<span class="hljs-property">userId</span>)!.<span class="hljs-property">name</span>);
<span class="hljs-comment">// ✅ O(n+m): dựng Map tra cứu O(1)</span>
<span class="hljs-keyword">const</span> nameById = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>(users.<span class="hljs-title function_">map</span>(<span class="hljs-function"><span class="hljs-params">u</span> =&gt;</span> [u.<span class="hljs-property">id</span>, u.<span class="hljs-property">name</span>]));
orders.<span class="hljs-title function_">forEach</span>(<span class="hljs-function"><span class="hljs-params">o</span> =&gt;</span> o.<span class="hljs-property">userName</span> = nameById.<span class="hljs-title function_">get</span>(o.<span class="hljs-property">userId</span>));
</code></pre>
<p><strong>Vì sao là mức ①:</strong> nắm khái niệm, chọn đúng cấu trúc cơ bản.</p>
<h2>▸ Ex·V1 — ② Biết làm</h2>
<p><strong>Khác Entry:</strong> áp OOP/pattern vào code thật để tách trách nhiệm; đọc được độ phức tạp để tránh code chậm ngầm.</p>
<p><strong>Ví dụ thực tế — Strategy pattern cho nhiều cách sắp xếp (thay vì if/else dài).</strong></p>
<pre><code class="language-typescript"><span class="hljs-keyword">type</span> <span class="hljs-title class_">Sorter</span> = <span class="hljs-function">(<span class="hljs-params"><span class="hljs-attr">a</span>: <span class="hljs-title class_">Order</span>, <span class="hljs-attr">b</span>: <span class="hljs-title class_">Order</span></span>) =&gt;</span> <span class="hljs-built_in">number</span>;

<p><strong>Vì sao là mức ②:</strong> vận dụng pattern để code sạch và đúng chi phí.</p>
<h2>▸ Ex·V2 — ② Biết làm (mở rộng phạm vi)</h2>
<p><strong>Khác V1:</strong> dùng cấu trúc/giải thuật cho bài toán FE thực tế — cây, đệ quy, gom nhóm — và biết đánh đổi.</p>
<p><strong>Ví dụ thực tế — dựng cây danh mục nhiều cấp từ dữ liệu phẳng.</strong></p>
<pre><code class="language-typescript"><span class="hljs-comment">// dữ liệu phẳng có parentId → dựng cây O(n) bằng Map thay vì lọc lặp O(n²)</span>

<p>Bạn nhận ra cách lọc lặp là O(n²) và biết khi nào nên đổi sang dựng Map một lần cho O(n).</p>
<p><strong>Vì sao vẫn là ②:</strong> xử lý tốt bài toán cấu trúc vừa, chưa tới tối ưu/giải thuật nâng cao.</p>
<h2>▸ Ex·V3 — ③ Thành thạo</h2>
<p><strong>Khác V2:</strong> nắm pattern ở mức kiến trúc, nhận diện và tối ưu điểm nghẽn giải thuật, chọn cấu trúc tối ưu có cân nhắc đánh đổi.</p>
<p><strong>Ví dụ 1 — tối ưu autocomplete 50.000 mục.</strong> Lọc bằng <code>includes</code> mỗi phím quá chậm. Bạn đánh giá phương án:</p>
<pre><code>- Index theo tiền tố / cấu trúc Trie → tra cứu nhanh hơn nhiều.
- Đánh đổi: Trie tốn bộ nhớ + dựng trước; hợp khi dữ liệu ổn định, truy vấn nhiều.
- Dữ liệu đổi liên tục → để server lo (full-text search) thay vì gánh ở client.
</code></pre>
<p><strong>Ví dụ 2 — memo hoá phép tính nặng lặp lại.</strong></p>
<pre><code class="language-typescript"><span class="hljs-keyword">const</span> cache = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Map</span>&lt;<span class="hljs-built_in">string</span>, <span class="hljs-title class_">Result</span>&gt;();

<p><strong>Vì sao là mức ③:</strong> dùng nền tảng CS để ra quyết định kiến trúc, không chỉ giải bài.</p>
<h2>▸ Senior·V3 — ④ Chuyên sâu</h2>
<p><strong>Khác Ex·V3:</strong> áp pattern ở <strong>tầm kiến trúc</strong> (nhiều module/nhiều nhà cung cấp), không chỉ trong phạm vi một class.</p>
