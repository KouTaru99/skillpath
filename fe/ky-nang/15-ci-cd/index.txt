18:T2ff0,<h1>CI/CD (Continuous Integration / Continuous Delivery)</h1>
<p><strong>Định nghĩa.</strong> <strong>CI</strong> (tích hợp liên tục): mỗi lần đẩy code, một quy trình tự động build, lint, test — bắt lỗi ngay. <strong>CD</strong> (chuyển giao/triển khai liên tục): tự động đóng gói và đưa code đã kiểm tra lên môi trường. Mục tiêu: mỗi thay đổi đi từ máy dev tới người dùng <strong>nhanh, đều, ít lỗi tay</strong>. Công cụ phổ biến: GitHub Actions, GitLab CI, Jenkins. Một FE dev phải đọc/sửa được pipeline phần của mình. Ví dụ dùng app Angular.</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu pipeline là gì, đọc được kết quả một lần chạy (fail ở bước nào), biết thay đổi phải qua các cổng (build → test → deploy).</p>
<p><strong>Ví dụ thực tế — pipeline Angular đơn giản (GitHub Actions).</strong></p>
<pre><code class="language-yaml"><span class="hljs-attr">name:</span> <span class="hljs-string">ci</span>
<span class="hljs-attr">on:</span> [<span class="hljs-string">push</span>]
<span class="hljs-attr">jobs:</span>
  <span class="hljs-attr">build:</span>
    <span class="hljs-attr">runs-on:</span> <span class="hljs-string">ubuntu-latest</span>
    <span class="hljs-attr">steps:</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">uses:</span> <span class="hljs-string">actions/checkout@v4</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">ci</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">lint</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">test</span> <span class="hljs-string">--</span> <span class="hljs-string">--watch=false</span> <span class="hljs-string">--browsers=ChromeHeadless</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">build</span> <span class="hljs-string">--</span> <span class="hljs-string">--configuration</span> <span class="hljs-string">production</span>
</code></pre>
<p>Pipeline đỏ ở bước <code>test</code> → mở log bước đó, đọc test nào fail, sửa rồi đẩy lại.</p>
<p><strong>Vì sao là mức ①:</strong> hiểu và đọc được pipeline, chưa tự cấu hình.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Khác V1:</strong> <strong>tự thêm/sửa bước</strong> cho phần FE, dùng cache để build nhanh, hiểu tách môi trường (preview/staging/production).</p>
<p><strong>Ví dụ 1 — cache dependency + deploy preview mỗi PR.</strong></p>
<pre><code class="language-yaml">      <span class="hljs-bullet">-</span> <span class="hljs-attr">uses:</span> <span class="hljs-string">actions/setup-node@v4</span>
        <span class="hljs-attr">with:</span> { <span class="hljs-attr">node-version:</span> <span class="hljs-number">18</span>, <span class="hljs-attr">cache:</span> <span class="hljs-string">&#x27;npm&#x27;</span> }   <span class="hljs-comment"># cache → build nhanh hơn</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">ci</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">lint</span> <span class="hljs-string">&amp;&amp;</span> <span class="hljs-string">npm</span> <span class="hljs-string">test</span> <span class="hljs-string">--</span> <span class="hljs-string">--watch=false</span> <span class="hljs-string">--browsers=ChromeHeadless</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">build</span> <span class="hljs-string">--</span> <span class="hljs-string">--configuration</span> <span class="hljs-string">production</span>
  <span class="hljs-attr">deploy-preview:</span>
    <span class="hljs-attr">needs:</span> <span class="hljs-string">build</span>              <span class="hljs-comment"># chỉ chạy khi build xanh</span>
    <span class="hljs-attr">if:</span> <span class="hljs-string">github.ref</span> <span class="hljs-type">!=</span> <span class="hljs-string">&#x27;refs/heads/main&#x27;</span>
    <span class="hljs-attr">steps:</span> [ { <span class="hljs-attr">run:</span> <span class="hljs-string">./deploy.sh</span> <span class="hljs-string">preview</span> } ]   <span class="hljs-comment"># mỗi PR có 1 bản preview để review thật</span>
</code></pre>
<p><strong>Ví dụ 2 — build theo môi trường (Angular file replacements).</strong> <code>environment.prod.ts</code> trỏ API production; CI build với <code>--configuration production</code> để Angular tự thay file môi trường — không hardcode URL.</p>
<p><strong>Vì sao là mức ②:</strong> vận hành và điều chỉnh pipeline phần mình phụ trách.</p>
<h2>▸ Ex·V3 — ② Biết làm (mở rộng phạm vi)</h2>
<p><strong>Khác V2:</strong> ở <strong>quy mô &amp; độ tin cậy cao hơn</strong> — tối ưu thời gian pipeline, thêm cổng chất lượng (coverage, kích thước bundle), chuẩn bị đường lùi (rollback).</p>
<p><strong>Ví dụ 1 — cổng chất lượng chặn hồi quy ngầm.</strong></p>
<pre><code class="language-yaml">      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">test</span> <span class="hljs-string">--</span> <span class="hljs-string">--code-coverage</span> <span class="hljs-string">--watch=false</span>   <span class="hljs-comment"># chặn nếu coverage tụt</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">build</span> <span class="hljs-string">--</span> <span class="hljs-string">--configuration</span> <span class="hljs-string">production</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npx</span> <span class="hljs-string">bundlesize</span>                              <span class="hljs-comment"># chặn nếu bundle phình quá ngưỡng</span>
</code></pre>
<p>Angular bundle dễ phình khi lỡ import cả thư viện lớn → cổng bundlesize bắt sớm.</p>
<p><strong>Ví dụ 2 — deploy an toàn có đường lùi.</strong></p>
<pre><code class="language-bash">deploy release-<span class="hljs-variable">$SHA</span> &amp;&amp; switch-traffic release-<span class="hljs-variable">$SHA</span>
<span class="hljs-comment"># nếu lỗi: switch-traffic release-$PREV   (quay lại trong vài giây)</span>
</code></pre>
<p>Bạn nghĩ tới &quot;khi deploy hỏng thì sao&quot; ngay từ đầu, không chỉ &quot;khi mọi thứ chạy tốt&quot;.</p>
<p><strong>Vì sao vẫn là ②:</strong> vận hành CI/CD vững ở quy mô thật, chưa tới thiết kế hạ tầng triển khai toàn hệ thống.</p>
<h2>▸ Senior·V1 — ③ Thành thạo</h2>
<p><strong>Khác Ex·V3:</strong> không chỉ tối ưu pipeline của dự án mình mà <strong>chuẩn hoá cho nhiều dự án/team dùng chung</strong> — đúng tinh thần &quot;thành thạo công cụ CI/CD&quot; ở tầm Senior trong career-path.</p>
<p><strong>Ví dụ — pipeline dùng chung (composite action) cho nhiều microfrontend khác team.</strong></p>
<pre><code class="language-yaml"><span class="hljs-comment"># .github/actions/fe-pipeline/action.yml — 1 chuẩn, nhiều repo MFE cùng gọi</span>
<span class="hljs-attr">runs:</span>
  <span class="hljs-attr">using:</span> <span class="hljs-string">composite</span>
  <span class="hljs-attr">steps:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">ci</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">lint</span> <span class="hljs-string">&amp;&amp;</span> <span class="hljs-string">npm</span> <span class="hljs-string">test</span> <span class="hljs-string">--</span> <span class="hljs-string">--code-coverage</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npx</span> <span class="hljs-string">bundlesize</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">npm</span> <span class="hljs-string">run</span> <span class="hljs-string">build</span> <span class="hljs-string">--</span> <span class="hljs-string">--configuration</span> <span class="hljs-string">production</span>
</code></pre>
<pre><code class="language-yaml"><span class="hljs-comment"># products-mfe/.github/workflows/ci.yml — mỗi team chỉ cần gọi lại</span>
<span class="hljs-attr">steps:</span> [ { <span class="hljs-attr">uses:</span> <span class="hljs-string">&#x27;./.github/actions/fe-pipeline&#x27;</span> } ]
</code></pre>
<p>Mỗi team vẫn tự chủ repo, nhưng chuẩn chất lượng (lint/test/coverage/bundle) là MỘT, không mỗi nơi làm một kiểu.</p>
<p><strong>Vì sao là mức ③:</strong> bạn nâng chất lượng CI/CD ở tầm nhiều dự án, không chỉ dự án đang làm.</p>
<h2>▸ Senior·V3 — ④ Chuyên sâu</h2>
<p><strong>Khác Senior·V1:</strong> thiết kế chiến lược <strong>deploy an toàn ở quy mô sản xuất thật</strong> — triển khai dần thay vì đổi toàn bộ một lúc.</p>
<p><strong>Ví dụ thực tế — canary release cho bản Angular mới, giảm rủi ro khi lên production.</strong></p>
<pre><code class="language-yaml"><span class="hljs-attr">deploy-canary:</span>
  <span class="hljs-attr">steps:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">deploy</span> <span class="hljs-string">new-version</span> <span class="hljs-string">--traffic</span> <span class="hljs-number">5</span><span class="hljs-string">%</span>     <span class="hljs-comment"># chỉ 5% người dùng thấy bản mới</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">sleep</span> <span class="hljs-number">600</span> <span class="hljs-string">&amp;&amp;</span> <span class="hljs-string">check-error-rate</span>        <span class="hljs-comment"># theo dõi 10 phút, so tỷ lệ lỗi với bản cũ</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">|
        if [ &quot;$ERROR_RATE_INCREASED&quot; = &quot;true&quot; ]; then
          rollback; exit 1                      # tự động lùi nếu lỗi tăng bất thường
        fi
</span>    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">deploy</span> <span class="hljs-string">new-version</span> <span class="hljs-string">--traffic</span> <span class="hljs-number">100</span><span class="hljs-string">%</span>   <span class="hljs-comment"># ổn thì mới đẩy full</span>
</code></pre>
<p>Nếu bản mới có bug ẩn (chỉ lộ ra ở một số trình duyệt/thiết bị cụ thể), chỉ 5% người dùng bị ảnh hưởng và hệ thống tự lùi lại — thay vì cả triệu người dùng cùng gặp lỗi rồi mới biết.</p>
<p><strong>Vì sao là mức ④:</strong> bạn thiết kế được quy trình triển khai giảm thiểu rủi ro ở quy mô thật, không chỉ làm cho pipeline chạy xanh.</p>
