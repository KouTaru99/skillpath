18:T2747,<h1>Component dùng chung (đóng gói &amp; tái sử dụng)</h1>
<p><strong>Định nghĩa.</strong> Component dùng chung là khối UI/logic được <strong>đóng gói để tái sử dụng</strong> nhiều nơi: nút, ô nhập, modal, bảng, hoặc directive/pipe. Mục tiêu <strong>DRY</strong> (Don't Repeat Yourself): viết một lần, dùng nhiều chỗ, sửa một chỗ là cập nhật toàn bộ. Làm tốt thì cả dự án đồng nhất; làm ẩu thì component &quot;dùng chung&quot; đầy ngoại lệ, khó dùng hơn tự viết. Ví dụ theo Angular.</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Tách khối lặp lại thành component nhận <code>@Input</code>, đặt API (tên input) rõ ràng.</p>
<p><strong>Ví dụ thực tế — nút dùng chung.</strong></p>
<pre><code class="language-typescript"><span class="hljs-meta">@Component</span>({
  <span class="hljs-attr">selector</span>: <span class="hljs-string">&#x27;app-button&#x27;</span>,
  <span class="hljs-attr">template</span>: <span class="hljs-string">`
    &lt;button [class]=&quot;&#x27;btn btn--&#x27; + variant&quot; [disabled]=&quot;loading&quot;&gt;
      &lt;app-spinner *ngIf=&quot;loading&quot; size=&quot;16&quot;&gt;&lt;/app-spinner&gt;
      &lt;ng-content *ngIf=&quot;!loading&quot;&gt;&lt;/ng-content&gt;
    &lt;/button&gt;`</span>,
})
<span class="hljs-keyword">export</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">ButtonComponent</span> {
  <span class="hljs-meta">@Input</span>() <span class="hljs-attr">variant</span>: <span class="hljs-string">&#x27;primary&#x27;</span> | <span class="hljs-string">&#x27;danger&#x27;</span> = <span class="hljs-string">&#x27;primary&#x27;</span>;   <span class="hljs-comment">// ràng buộc giá trị, tránh truyền bừa</span>
  <span class="hljs-meta">@Input</span>() loading = <span class="hljs-literal">false</span>;
}
<span class="hljs-comment">// Dùng: &lt;app-button variant=&quot;danger&quot; [loading]=&quot;deleting&quot;&gt;Xoá&lt;/app-button&gt;</span>
</code></pre>
<p><code>&lt;ng-content&gt;</code> cho phép nhét nội dung tuỳ ý vào (content projection).</p>
<p><strong>Vì sao là mức ①:</strong> tách được component tái dùng đơn giản, chưa lường hết tình huống dùng.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Khác V1:</strong> component <strong>linh hoạt mà nhất quán</strong> — API hợp lý, mặc định tốt, kiểm soát kiểu (TypeScript), a11y, tránh &quot;input bùng nổ&quot;.</p>
<p><strong>Ví dụ 1 — Modal tái dùng đúng cách (Esc để đóng, role dialog).</strong></p>
<pre><code class="language-typescript"><span class="hljs-meta">@Component</span>({ <span class="hljs-attr">selector</span>: <span class="hljs-string">&#x27;app-modal&#x27;</span>, <span class="hljs-attr">template</span>: <span class="hljs-string">`
  &lt;div class=&quot;overlay&quot; (click)=&quot;close.emit()&quot;&gt;
    &lt;div class=&quot;modal&quot; role=&quot;dialog&quot; [attr.aria-label]=&quot;title&quot; (click)=&quot;$event.stopPropagation()&quot;&gt;
      &lt;h2&gt;{{ title }}&lt;/h2&gt;
      &lt;ng-content&gt;&lt;/ng-content&gt;
    &lt;/div&gt;
  &lt;/div&gt;`</span> })
<span class="hljs-keyword">export</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">ModalComponent</span> {
  <span class="hljs-meta">@Input</span>() title = <span class="hljs-string">&#x27;&#x27;</span>;
  <span class="hljs-meta">@Output</span>() close = <span class="hljs-keyword">new</span> <span class="hljs-title class_">EventEmitter</span>&lt;<span class="hljs-built_in">void</span>&gt;();
  <span class="hljs-meta">@HostListener</span>(<span class="hljs-string">&#x27;document:keydown.escape&#x27;</span>) <span class="hljs-title function_">onEsc</span>(<span class="hljs-params"></span>) { <span class="hljs-variable language_">this</span>.<span class="hljs-property">close</span>.<span class="hljs-title function_">emit</span>(); }
}
</code></pre>
<p><strong>Ví dụ 2 — directive dùng chung (highlight khi hover).</strong> Không phải cái gì cũng là component — logic gắn vào phần tử thì dùng directive:</p>
<pre><code class="language-typescript"><span class="hljs-meta">@Directive</span>({ <span class="hljs-attr">selector</span>: <span class="hljs-string">&#x27;[appHighlight]&#x27;</span> })

<p><strong>Vì sao là mức ②:</strong> component bền, hợp đồng rõ, người khác dùng không cần đọc mã nguồn.</p>
<h2>▸ Ex·V3 — ③ Thành thạo</h2>
<p><strong>Khác V2:</strong> xây <strong>bộ component cấp dự án/công ty</strong> — nhất quán Design System, có tài liệu/demo, cân nhắc đánh đổi linh hoạt vs đơn giản.</p>
<p><strong>Ví dụ 1 — bảng dữ liệu tái dùng với template tuỳ biến cột (ng-template + context).</strong></p>
<pre><code class="language-typescript"><span class="hljs-meta">@Component</span>({ <span class="hljs-attr">selector</span>: <span class="hljs-string">&#x27;app-table&#x27;</span>, <span class="hljs-attr">template</span>: <span class="hljs-string">`
  &lt;tr *ngFor=&quot;let row of rows&quot;&gt;
    &lt;td *ngFor=&quot;let col of columns&quot;&gt;
      &lt;ng-container *ngTemplateOutlet=&quot;col.cell || default; context: { $implicit: row }&quot;&gt;
      &lt;/ng-container&gt;
    &lt;/td&gt;
  &lt;/tr&gt;`</span> })
<span class="hljs-keyword">export</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">TableComponent</span> { <span class="hljs-meta">@Input</span>() rows; <span class="hljs-meta">@Input</span>() columns; }
<span class="hljs-comment">// Người dùng tự định nghĩa cách render từng cột → bảng linh hoạt mà không cần 20 input</span>
</code></pre>
<p><strong>Ví dụ 2 — pipe dùng chung (định dạng tiền VND).</strong></p>
<pre><code class="language-typescript"><span class="hljs-meta">@Pipe</span>({ <span class="hljs-attr">name</span>: <span class="hljs-string">&#x27;vnd&#x27;</span> })
<span class="hljs-keyword">export</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">VndPipe</span> <span class="hljs-keyword">implements</span> <span class="hljs-title class_">PipeTransform</span> {

<span class="hljs-comment">// {{ order.total | vnd }} → &quot;1.250.000₫&quot;</span>
</code></pre>
<p><strong>Ví dụ 3 — đóng gói thành shared module + tài liệu sống.</strong> Gom vào <code>SharedModule</code>, viết demo (Storybook) làm tài liệu, đặt quy ước versioning để team khác nâng cấp an toàn.</p>
<p><strong>Vì sao là mức ③:</strong> tạo nền tảng dùng chung cho nhiều người, không chỉ cho dự án của mình.</p>
<h2>▸ Senior·V2 — ④ Chuyên sâu</h2>
<p><strong>Khác Ex·V3:</strong> không chỉ <em>xây</em> component dùng chung mà <strong>quản lý vòng đời của nó khi đã có nhiều team phụ thuộc</strong> — đổi API mà không làm sập dự án khác.</p>
<p><strong>Ví dụ thực tế — đổi API của <code>TableComponent</code> mà 6 dự án khác đang dùng.</strong> Bạn cần đổi <code>columns</code> từ mảng object sang cấu hình mạnh hơn (hỗ trợ sort/filter), nhưng phá thẳng API cũ sẽ làm sập build của 6 dự án đang phụ thuộc. Cách xử lý đúng tầm Senior:</p>
<pre><code class="language-typescript"><span class="hljs-meta">@Input</span>() columns!: <span class="hljs-title class_">Column</span>[];              <span class="hljs-comment">// API cũ — GIỮ NGUYÊN, đánh dấu deprecated</span>
<span class="hljs-comment">/** <span class="hljs-doctag">@deprecated</span> dùng `columnsConfig` từ v3.0, sẽ gỡ ở v4.0 — xem CHANGELOG */</span>
<span class="hljs-meta">@Input</span>() <span class="hljs-attr">columnsConfig</span>?: <span class="hljs-title class_">ColumnConfig</span>[];   <span class="hljs-comment">// API mới, opt-in — team nào sẵn sàng thì đổi</span>
</code></pre>
<p>Kèm theo <code>CHANGELOG.md</code> ghi rõ mốc gỡ bỏ, và một major version (<code>v3.0.0</code> theo <strong>semver</strong>) để các team chủ động chọn thời điểm nâng cấp thay vì bị ép đổi ngay.</p>
<p><strong>Vì sao là mức ④:</strong> bạn quản trị được thư viện dùng chung <strong>qua thời gian</strong> — cho phép nó tiến hoá mà không phá vỡ những đội đang phụ thuộc vào nó, đúng bản chất hạ tầng cấp công ty.</p>
