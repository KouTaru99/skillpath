18:T21e0,<h1>Docker (đóng gói ứng dụng)</h1>
<p><strong>Định nghĩa.</strong> <strong>Docker</strong> đóng gói ứng dụng cùng mọi thứ nó cần (mã, runtime, thư viện, cấu hình) vào một <strong>image</strong> — chạy lên thành <strong>container</strong> giống hệt nhau ở mọi máy. Giải quyết bệnh <em>&quot;máy em chạy được mà&quot;</em>. Khác máy ảo ở chỗ container nhẹ, dùng chung nhân hệ điều hành nên khởi động vài giây. Với FE, Docker thường dùng để build và phục vụ app tĩnh (vd Angular build ra file tĩnh rồi Nginx serve).</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu image vs container, đọc/viết <code>Dockerfile</code> cơ bản, build/run/expose port một app.</p>
<p><strong>Ví dụ thực tế — đóng gói app Angular (build tĩnh + Nginx).</strong></p>
<pre><code class="language-dockerfile"><span class="hljs-keyword">FROM</span> node:<span class="hljs-number">18</span>-alpine
<span class="hljs-keyword">WORKDIR</span><span class="language-bash"> /app</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> package*.json ./</span>
<span class="hljs-keyword">RUN</span><span class="language-bash"> npm ci</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> . .</span>
<span class="hljs-keyword">RUN</span><span class="language-bash"> npm run build          <span class="hljs-comment"># ra thư mục dist/&lt;app&gt;/</span></span>
<span class="hljs-keyword">EXPOSE</span> <span class="hljs-number">4200</span>
<span class="hljs-keyword">CMD</span><span class="language-bash"> [<span class="hljs-string">&quot;npx&quot;</span>, <span class="hljs-string">&quot;http-server&quot;</span>, <span class="hljs-string">&quot;dist/my-app&quot;</span>, <span class="hljs-string">&quot;-p&quot;</span>, <span class="hljs-string">&quot;4200&quot;</span>]</span>
</code></pre>
<pre><code class="language-bash">docker build -t my-fe .            <span class="hljs-comment"># tạo image (khuôn bất biến)</span>
docker run -p 4200:4200 my-fe      <span class="hljs-comment"># chạy container (thực thể)</span>
</code></pre>
<p><strong>Vì sao là mức ①:</strong> đóng gói và chạy được, chưa tối ưu image.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Khác V1:</strong> viết <code>Dockerfile</code> <strong>tối ưu</strong> (multi-stage để image nhỏ, tận dụng cache layer), dùng <code>docker compose</code> chạy nhiều dịch vụ khi dev.</p>
<p><strong>Ví dụ 1 — multi-stage: build bằng Node, chạy bằng Nginx (image gọn, không kéo theo Node).</strong></p>
<pre><code class="language-dockerfile"><span class="hljs-comment"># Pha 1: build</span>
<span class="hljs-keyword">FROM</span> node:<span class="hljs-number">18</span>-alpine AS build
<span class="hljs-keyword">WORKDIR</span><span class="language-bash"> /app</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> package*.json ./</span>
<span class="hljs-keyword">RUN</span><span class="language-bash"> npm ci</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> . .</span>
<span class="hljs-keyword">RUN</span><span class="language-bash"> npm run build -- --configuration production</span>

<span class="hljs-comment"># Pha 2: chỉ lấy file tĩnh, phục vụ bằng Nginx</span>
<span class="hljs-keyword">FROM</span> nginx:alpine
<span class="hljs-keyword">COPY</span><span class="language-bash"> --from=build /app/dist/my-app /usr/share/nginx/html</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> nginx.conf /etc/nginx/conf.d/default.conf   <span class="hljs-comment"># cấu hình SPA fallback về index.html</span></span>
<span class="hljs-keyword">EXPOSE</span> <span class="hljs-number">80</span>
</code></pre>
<p>Copy <code>package.json</code> trước rồi <code>npm ci</code>, sau đó mới copy code → đổi code không phải cài lại dependency (tận dụng cache layer).</p>
<p><strong>Ví dụ 2 — chạy cả FE + API + DB khi dev bằng một lệnh.</strong></p>
<pre><code class="language-yaml"><span class="hljs-comment"># docker-compose.yml</span>
<span class="hljs-attr">services:</span>
  <span class="hljs-attr">web:</span> { <span class="hljs-attr">build:</span> <span class="hljs-string">.</span>, <span class="hljs-attr">ports:</span> [<span class="hljs-string">&quot;8080:80&quot;</span>] }
  <span class="hljs-attr">api:</span> { <span class="hljs-attr">image:</span> <span class="hljs-string">my-spring-api</span>, <span class="hljs-attr">ports:</span> [<span class="hljs-string">&quot;3000:3000&quot;</span>] }
  <span class="hljs-attr">db:</span>  { <span class="hljs-attr">image:</span> <span class="hljs-string">postgres:16</span>, <span class="hljs-attr">environment:</span> { <span class="hljs-attr">POSTGRES_PASSWORD:</span> <span class="hljs-string">dev</span> } }
</code></pre>
<p><strong>Vì sao là mức ②:</strong> đóng gói gọn, đúng chuẩn, chạy được môi trường nhiều dịch vụ.</p>
<h2>▸ Ex·V3 — ② Biết làm (mở rộng phạm vi)</h2>
<p><strong>Khác V2:</strong> quan tâm <strong>kích thước, bảo mật và tích hợp CI/CD</strong> — image nhỏ và sạch, không nhúng secret, chạy bằng user không phải root.</p>
<p><strong>Ví dụ 1 — SPA fallback + không lộ secret.</strong></p>
<pre><code class="language-nginx"><span class="hljs-comment"># nginx.conf — mọi route trả index.html để Angular router xử lý</span>
<span class="hljs-section">location</span> / { <span class="hljs-attribute">try_files</span> <span class="hljs-variable">$uri</span> <span class="hljs-variable">$uri</span>/ /index.html; }
</code></pre>
<p>Cấu hình API URL truyền lúc chạy (biến môi trường / file config), <strong>không build cứng secret vào image</strong>.</p>
<p><strong>Ví dụ 2 — dùng <code>.dockerignore</code> + ghim version cho build lặp lại được.</strong></p>
<pre><code># .dockerignore
node_modules
dist
.git
.env
</code></pre>
<p>Ghim <code>node:18-alpine</code> (không dùng <code>latest</code>) để build hôm nay và tháng sau ra giống nhau; để CI tự build–đẩy image lên registry.</p>
<p><strong>Vì sao vẫn là ②:</strong> đóng gói tốt và an toàn ở mức thực chiến, chưa tới thiết kế nền tảng container cho tổ chức (việc của Senior/Specialist).</p>
<h2>▸ Senior·V1 — ③ Thành thạo</h2>
<p><strong>Khác Ex·V3:</strong> không chỉ đóng gói an toàn cho <em>một</em> app mà đặt ra <strong>base image chuẩn dùng chung cho nhiều dự án</strong>, kèm healthcheck để hạ tầng biết container còn sống hay đã &quot;treo&quot;.</p>
<p><strong>Ví dụ — base image nội bộ (golden image) + healthcheck cho orchestration.</strong></p>
<pre><code class="language-dockerfile"><span class="hljs-comment"># fe-base-image (registry nội bộ) — mọi FE team kế thừa từ đây, không tự chọn base image riêng</span>
<span class="hljs-keyword">FROM</span> nginx:alpine
<span class="hljs-keyword">RUN</span><span class="language-bash"> adduser -D -H appuser &amp;&amp; <span class="hljs-built_in">chown</span> -R appuser /usr/share/nginx/html</span>
<span class="hljs-keyword">USER</span> appuser                                   <span class="hljs-comment"># không chạy container bằng root</span>
<span class="hljs-keyword">HEALTHCHECK</span><span class="language-bash"> --interval=30s --<span class="hljs-built_in">timeout</span>=3s \
  CMD wget -q --spider http://localhost/ || <span class="hljs-built_in">exit</span> 1   <span class="hljs-comment"># hạ tầng tự phát hiện + restart nếu &quot;treo&quot;</span></span>
</code></pre>
<pre><code class="language-dockerfile"><span class="hljs-comment"># products-mfe/Dockerfile — team chỉ kế thừa, không tự nghĩ lại phần bảo mật</span>
<span class="hljs-keyword">FROM</span> registry.internal/fe-base-image:<span class="hljs-number">1.4</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> --from=build /app/dist /usr/share/nginx/html</span>
</code></pre>
<p>Chuẩn hoá base image giúp mọi team tự động thừa hưởng bản vá bảo mật khi image gốc cập nhật, thay vì mỗi repo tự vá một kiểu (hoặc quên vá).</p>
<p><strong>Vì sao là mức ③:</strong> bạn đặt chuẩn container cho tổ chức, không chỉ đóng gói đúng cho dự án của mình.</p>
