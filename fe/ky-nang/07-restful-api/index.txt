18:T399e,<h1>RESTful API (tích hợp Back-end)</h1>
<p><strong>Định nghĩa.</strong> <strong>API</strong> là &quot;hợp đồng&quot; để hai phần mềm nói chuyện. <strong>REST</strong> là phong cách thiết kế API qua HTTP: mỗi <em>tài nguyên</em> có URL, thao tác bằng <strong>method</strong> (<code>GET</code> đọc, <code>POST</code> tạo, <code>PUT/PATCH</code> sửa, <code>DELETE</code> xoá) + <strong>status code</strong> chuẩn (2xx ok, 4xx lỗi client, 5xx lỗi server). Trang này cho thấy <strong>cả hai đầu</strong>: Angular gọi API (<code>HttpClient</code>) và phía server trả gì (<strong>Java/Spring Boot</strong>) — để hiểu đúng hợp đồng.</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu method/URL/status code, gọi CRUD cơ bản bằng <code>HttpClient</code>, đọc kết quả, xử lý nhánh thành công/lỗi.</p>
<p><strong>Ví dụ thực tế — CRUD một tài nguyên (Angular).</strong></p>
<pre><code class="language-typescript"><span class="hljs-variable language_">this</span>.<span class="hljs-property">http</span>.<span class="hljs-property">get</span>&lt;<span class="hljs-title class_">User</span>[]&gt;(<span class="hljs-string">&#x27;/api/users&#x27;</span>);                          <span class="hljs-comment">// GET 200</span>
<span class="hljs-variable language_">this</span>.<span class="hljs-property">http</span>.<span class="hljs-property">post</span>&lt;<span class="hljs-title class_">User</span>&gt;(<span class="hljs-string">&#x27;/api/users&#x27;</span>, { name, email });          <span class="hljs-comment">// POST 201</span>

<p>Phía server (Spring Boot) — để hiểu 201/204 từ đâu ra:</p>

<p><strong>Vì sao là mức ①:</strong> gọi đúng và đọc kết quả, chưa xử lý lỗi/đồng bộ bài bản.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Xử lý <strong>đầy đủ vòng đời request</strong> — phân loại lỗi theo status, gửi token, phân trang/lọc qua query param, đồng bộ UI với kết quả.</p>

<p><strong>Vì sao là mức ②:</strong> tích hợp API đúng bài, chủ động với lỗi và xác thực.</p>
<h2>▸ Ex·V3 — ② Biết làm (mở rộng phạm vi)</h2>
<p><strong>Khác V2:</strong> ở <strong>quy mô tích hợp nhiều API</strong> — gọi song song, gộp/biến đổi dữ liệu, chống race, đặt lớp API dùng chung.</p>
<p><strong>Ví dụ 1 — dashboard gọi nhiều API song song (<code>forkJoin</code>).</strong></p>

<p><strong>Vì sao vẫn là ②:</strong> tích hợp vững ở quy mô thật, chưa tới mức thiết kế chuẩn giao tiếp cấp hệ thống.</p>
<h2>▸ Senior·V1 — ③ Thành thạo</h2>
<p><strong>Khác Ex·V3:</strong> không chỉ <em>gọi</em> API tốt mà <strong>đặt ra chuẩn giao tiếp</strong> cho cả dự án — envelope lỗi thống nhất, versioning, hợp đồng viết trước bằng OpenAPI để BE/FE làm song song không lệch nhau.</p>
<p><strong>Ví dụ 1 — envelope lỗi thống nhất, mọi API trả cùng khuôn.</strong></p>
<pre><code class="language-typescript"><span class="hljs-keyword">interface</span> <span class="hljs-title class_">ApiError</span> { <span class="hljs-attr">code</span>: <span class="hljs-built_in">string</span>; <span class="hljs-attr">message</span>: <span class="hljs-built_in">string</span>; <span class="hljs-attr">fields</span>?: <span class="hljs-title class_">Record</span>&lt;<span class="hljs-built_in">string</span>, <span class="hljs-built_in">string</span>&gt; }

<span class="hljs-variable language_">this</span>.<span class="hljs-property">http</span>.<span class="hljs-title function_">get</span>(<span class="hljs-string">&#x27;/api/orders&#x27;</span>).<span class="hljs-title function_">pipe</span>(
  <span class="hljs-title function_">catchError</span>(<span class="hljs-function">(<span class="hljs-params"><span class="hljs-attr">e</span>: <span class="hljs-title class_">HttpErrorResponse</span></span>) =&gt;</span> <span class="hljs-title function_">throwError</span>(<span class="hljs-function">() =&gt;</span> e.<span class="hljs-property">error</span> <span class="hljs-keyword">as</span> <span class="hljs-title class_">ApiError</span>)),
);
</code></pre>
<pre><code class="language-java"><span class="hljs-meta">@ExceptionHandler(ValidationException.class)</span>

  <span class="hljs-keyword">return</span> ResponseEntity.badRequest().body(<span class="hljs-keyword">new</span> <span class="hljs-title class_">ApiError</span>(<span class="hljs-string">&quot;VALIDATION_FAILED&quot;</span>, ex.getMessage(), ex.getFieldErrors()));

</code></pre>
<p><strong>Ví dụ 2 — hợp đồng viết trước (OpenAPI) để FE/BE làm song song.</strong> Trước khi BE code xong, hai bên chốt file <code>orders.yaml</code> mô tả request/response — FE sinh type từ đó (<code>openapi-typescript</code>) và mock server chạy ngay, không phải chờ BE.</p>
<p><strong>Ví dụ 3 — versioning khi đổi hợp đồng.</strong> Đổi field <code>customerName</code> → tách <code>firstName</code>/<code>lastName</code> sẽ vỡ app cũ đang chạy → mở <code>/api/v2/orders</code> song song <code>/api/v1/orders</code>, tắt dần v1 theo lộ trình thay vì sửa thẳng.</p>
<p><strong>Vì sao là mức ③:</strong> bạn không chỉ tiêu thụ API mà <strong>định hình</strong> cách cả team giao tiếp qua API, giảm việc &quot;đoán&quot; giữa FE và BE.</p>
<h2>▸ Senior·V3 — ④ Chuyên sâu</h2>
<p><strong>Khác Senior·V1:</strong> đặt ra <strong>nguyên tắc thiết kế API cho toàn công ty</strong> (không chỉ một dự án) — ví dụ chuẩn hoá <strong>idempotency</strong> (gọi lại nhiều lần không gây tác dụng phụ kép) cho các API tạo mới.</p>
<p><strong>Ví dụ thực tế — API &quot;tạo đơn hàng&quot; bị gọi 2 lần do mạng chập chờn, và cách thiết kế để an toàn.</strong></p>
<pre><code class="language-java"><span class="hljs-meta">@PostMapping(&quot;/api/orders&quot;)</span>
<span class="hljs-keyword">public</span> ResponseEntity&lt;Order&gt; <span class="hljs-title function_">create</span><span class="hljs-params">(<span class="hljs-meta">@RequestHeader(&quot;Idempotency-Key&quot;)</span> String key, <span class="hljs-meta">@RequestBody</span> OrderDto dto)</span> {
  Optional&lt;Order&gt; existing = repo.findByIdempotencyKey(key);
  <span class="hljs-keyword">if</span> (existing.isPresent()) <span class="hljs-keyword">return</span> ResponseEntity.ok(existing.get());  <span class="hljs-comment">// gọi lại → trả kết quả cũ, KHÔNG tạo đơn mới</span>
  <span class="hljs-type">Order</span> <span class="hljs-variable">saved</span> <span class="hljs-operator">=</span> repo.save(<span class="hljs-keyword">new</span> <span class="hljs-title class_">Order</span>(dto, key));
  <span class="hljs-keyword">return</span> ResponseEntity.status(HttpStatus.CREATED).body(saved);
}
