18:T21f4,<h1>CI/CD (Continuous Integration / Continuous Delivery)</h1>
<p><strong>Định nghĩa.</strong> <strong>CI</strong> (tích hợp liên tục): mỗi lần đẩy code, một quy trình tự động build, test, kiểm tra chất lượng — bắt lỗi ngay. <strong>CD</strong> (chuyển giao liên tục): tự động đóng gói và đưa code lên môi trường. Một Back-end dev phải đọc/sửa được pipeline phần của mình.</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu pipeline là gì, đọc được kết quả một lần chạy (fail ở bước nào).</p>
<p><strong>Ví dụ thực tế — pipeline Java/Spring Boot đơn giản (GitHub Actions).</strong></p>
<pre><code class="language-yaml"><span class="hljs-attr">name:</span> <span class="hljs-string">ci</span>
<span class="hljs-attr">on:</span> [<span class="hljs-string">push</span>]
<span class="hljs-attr">jobs:</span>
  <span class="hljs-attr">build:</span>
    <span class="hljs-attr">runs-on:</span> <span class="hljs-string">ubuntu-latest</span>
    <span class="hljs-attr">steps:</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">uses:</span> <span class="hljs-string">actions/checkout@v4</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">uses:</span> <span class="hljs-string">actions/setup-java@v4</span>
        <span class="hljs-attr">with:</span> { <span class="hljs-attr">java-version:</span> <span class="hljs-string">&#x27;17&#x27;</span>, <span class="hljs-attr">distribution:</span> <span class="hljs-string">&#x27;temurin&#x27;</span> }
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">./gradlew</span> <span class="hljs-string">test</span>        <span class="hljs-comment"># chạy toàn bộ unit test</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">./gradlew</span> <span class="hljs-string">build</span>       <span class="hljs-comment"># build ra file .jar</span>
</code></pre>
<p>Pipeline đỏ ở bước <code>test</code> → mở log, đọc test nào fail, sửa rồi đẩy lại — không phải chỉ chạy <code>./gradlew build</code> trên máy mình rồi coi là xong.</p>
<p><strong>Vì sao là mức ①:</strong> hiểu và đọc được pipeline cơ bản — chưa tự cấu hình hay tối ưu pipeline.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Khác V1:</strong> tự thêm/sửa bước cho pipeline (cache dependency, build image Docker, deploy).</p>
<p><strong>Ví dụ thực tế — cache dependency + build &amp; push Docker image.</strong></p>
<pre><code class="language-yaml">      <span class="hljs-bullet">-</span> <span class="hljs-attr">uses:</span> <span class="hljs-string">actions/setup-java@v4</span>
        <span class="hljs-attr">with:</span> { <span class="hljs-attr">java-version:</span> <span class="hljs-string">&#x27;17&#x27;</span>, <span class="hljs-attr">distribution:</span> <span class="hljs-string">&#x27;temurin&#x27;</span>, <span class="hljs-attr">cache:</span> <span class="hljs-string">&#x27;gradle&#x27;</span> }   <span class="hljs-comment"># cache → build nhanh hơn</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">./gradlew</span> <span class="hljs-string">test</span> <span class="hljs-string">build</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">docker</span> <span class="hljs-string">build</span> <span class="hljs-string">-t</span> <span class="hljs-string">registry.internal/order-service:${{</span> <span class="hljs-string">github.sha</span> <span class="hljs-string">}}</span> <span class="hljs-string">.</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">docker</span> <span class="hljs-string">push</span> <span class="hljs-string">registry.internal/order-service:${{</span> <span class="hljs-string">github.sha</span> <span class="hljs-string">}}</span>
</code></pre>
<p>Bạn tự thêm bước build/push image Docker sau khi test xanh — ghim tag theo commit SHA để biết chính xác image nào ứng với commit nào.</p>
<p><strong>Vì sao là mức ②:</strong> vận hành và điều chỉnh pipeline phần mình phụ trách — không chỉ đọc kết quả.</p>
<h2>▸ Senior·V2 — ③ Thành thạo</h2>
<p><strong>Khác Ex·V2:</strong> sử dụng thành thạo CI/CD ở mức chuẩn hoá — pipeline dùng chung cho nhiều service, gate chất lượng (coverage, security scan).</p>
<p><strong>Ví dụ thực tế — pipeline dùng chung (reusable workflow) cho nhiều service Java.</strong></p>
<pre><code class="language-yaml"><span class="hljs-comment"># .github/workflows/java-service-ci.yml — 1 chuẩn, nhiều service cùng gọi</span>
<span class="hljs-attr">on:</span> { <span class="hljs-attr">workflow_call:</span> {} }
<span class="hljs-attr">jobs:</span>
  <span class="hljs-attr">build:</span>
    <span class="hljs-attr">steps:</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">./gradlew</span> <span class="hljs-string">test</span> <span class="hljs-string">build</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">./gradlew</span> <span class="hljs-string">jacocoTestCoverageVerification</span>   <span class="hljs-comment"># chặn nếu coverage tụt</span>
      <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">trivy</span> <span class="hljs-string">image</span> <span class="hljs-string">${{</span> <span class="hljs-string">github.repository</span> <span class="hljs-string">}}</span>         <span class="hljs-comment"># quét lỗ hổng bảo mật image</span>
</code></pre>
<pre><code class="language-yaml"><span class="hljs-comment"># order-service/.github/workflows/ci.yml — mỗi service chỉ cần gọi lại</span>
<span class="hljs-attr">jobs:</span>
  <span class="hljs-attr">ci:</span>
    <span class="hljs-attr">uses:</span> <span class="hljs-string">./.github/workflows/java-service-ci.yml</span>
</code></pre>
<p>Mỗi service vẫn tự chủ repo, nhưng chuẩn chất lượng (test/coverage/security scan) là MỘT, không mỗi nơi làm một kiểu.</p>
<p><strong>Vì sao là mức ③:</strong> bạn nâng chất lượng CI/CD ở tầm nhiều service, không chỉ dự án đang làm.</p>
<h2>▸ Specialist·V2 — ④ Chuyên sâu</h2>
<p><strong>Khác Senior·V2:</strong> thiết kế chiến lược <strong>deploy an toàn ở quy mô sản xuất thật</strong> — triển khai dần thay vì đổi toàn bộ một lúc.</p>
<p><strong>Ví dụ thực tế — canary release cho bản service mới, giảm rủi ro khi lên production.</strong></p>
<pre><code class="language-yaml"><span class="hljs-attr">deploy-canary:</span>
  <span class="hljs-attr">steps:</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">deploy</span> <span class="hljs-string">new-version</span> <span class="hljs-string">--traffic</span> <span class="hljs-number">5</span><span class="hljs-string">%</span>     <span class="hljs-comment"># chỉ 5% traffic thấy bản mới</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">sleep</span> <span class="hljs-number">600</span> <span class="hljs-string">&amp;&amp;</span> <span class="hljs-string">check-error-rate</span>        <span class="hljs-comment"># theo dõi 10 phút</span>
    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">|
        if [ &quot;$ERROR_RATE_INCREASED&quot; = &quot;true&quot; ]; then rollback; exit 1; fi
</span>    <span class="hljs-bullet">-</span> <span class="hljs-attr">run:</span> <span class="hljs-string">deploy</span> <span class="hljs-string">new-version</span> <span class="hljs-string">--traffic</span> <span class="hljs-number">100</span><span class="hljs-string">%</span>
</code></pre>
<p>Nếu bản mới có bug ẩn, chỉ 5% người dùng bị ảnh hưởng và hệ thống tự lùi lại — thay vì cả hệ thống cùng gặp lỗi rồi mới biết.</p>
<p><strong>Vì sao là mức ④:</strong> bạn thiết kế được quy trình triển khai giảm thiểu rủi ro ở quy mô thật — mức cao nhất của kỹ năng này.</p>
