18:T1642,<h1>Docker (đóng gói ứng dụng)</h1>
<p><strong>Định nghĩa.</strong> <strong>Docker</strong> đóng gói ứng dụng cùng mọi thứ nó cần (mã, runtime, thư viện, cấu hình) vào một <strong>image</strong> — chạy lên thành <strong>container</strong> giống hệt nhau ở mọi máy. Giải quyết bệnh <em>&quot;máy em chạy được mà&quot;</em>. Với Back-end, Docker thường dùng để đóng gói service Java cùng JVM cần thiết.</p>
<h2>▸ Ex·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu nguyên lý image vs container, build/run image, expose port.</p>
<p><strong>Ví dụ thực tế — đóng gói một service Spring Boot.</strong></p>
<pre><code class="language-dockerfile"><span class="hljs-keyword">FROM</span> eclipse-temurin:<span class="hljs-number">17</span>-jre-alpine
<span class="hljs-keyword">WORKDIR</span><span class="language-bash"> /app</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> target/order-service.jar app.jar</span>
<span class="hljs-keyword">EXPOSE</span> <span class="hljs-number">8080</span>
<span class="hljs-keyword">CMD</span><span class="language-bash"> [<span class="hljs-string">&quot;java&quot;</span>, <span class="hljs-string">&quot;-jar&quot;</span>, <span class="hljs-string">&quot;app.jar&quot;</span>]</span>
</code></pre>
<pre><code class="language-bash">docker build -t order-service .          <span class="hljs-comment"># tạo image (khuôn bất biến)</span>
docker run -p 8080:8080 order-service    <span class="hljs-comment"># chạy container (thực thể), map port 8080</span>
</code></pre>
<p>Bạn hiểu <code>EXPOSE 8080</code> khai báo container sẽ lắng nghe cổng nào, và <code>-p 8080:8080</code> khi <code>run</code> là ánh xạ cổng đó ra máy thật để gọi được từ bên ngoài.</p>
<p><strong>Vì sao là mức ①:</strong> đóng gói và chạy được một service đơn giản — chưa tối ưu image (multi-stage) hay dùng docker compose cho nhiều service.</p>
<h2>▸ Ex·V2 — ② Biết làm</h2>
<p><strong>Khác V1:</strong> viết Dockerfile <strong>multi-stage</strong> (image gọn hơn) và chạy nhiều service cùng lúc bằng <strong>docker compose</strong>.</p>
<p><strong>Ví dụ 1 — multi-stage: build bằng Gradle, chạy chỉ với JRE (không kéo theo cả bộ build).</strong></p>
<pre><code class="language-dockerfile"><span class="hljs-keyword">FROM</span> gradle:<span class="hljs-number">8</span>-jdk17 AS build
<span class="hljs-keyword">WORKDIR</span><span class="language-bash"> /app</span>
<span class="hljs-keyword">COPY</span><span class="language-bash"> . .</span>
<span class="hljs-keyword">RUN</span><span class="language-bash"> gradle build -x <span class="hljs-built_in">test</span></span>

<span class="hljs-keyword">FROM</span> eclipse-temurin:<span class="hljs-number">17</span>-jre-alpine
<span class="hljs-keyword">COPY</span><span class="language-bash"> --from=build /app/build/libs/order-service.jar app.jar</span>
<span class="hljs-keyword">CMD</span><span class="language-bash"> [<span class="hljs-string">&quot;java&quot;</span>, <span class="hljs-string">&quot;-jar&quot;</span>, <span class="hljs-string">&quot;app.jar&quot;</span>]</span>
</code></pre>
<p><strong>Ví dụ 2 — chạy service + CSDL bằng một lệnh khi dev.</strong></p>
<pre><code class="language-yaml"><span class="hljs-comment"># docker-compose.yml</span>
<span class="hljs-attr">services:</span>
  <span class="hljs-attr">api:</span> { <span class="hljs-attr">build:</span> <span class="hljs-string">.</span>, <span class="hljs-attr">ports:</span> [<span class="hljs-string">&quot;8080:8080&quot;</span>] }
  <span class="hljs-attr">db:</span>  { <span class="hljs-attr">image:</span> <span class="hljs-string">postgres:16</span>, <span class="hljs-attr">environment:</span> { <span class="hljs-attr">POSTGRES_PASSWORD:</span> <span class="hljs-string">dev</span> } }
</code></pre>
<p><strong>Vì sao là mức ②:</strong> đóng gói gọn (multi-stage) và chạy được môi trường nhiều dịch vụ — không chỉ đóng gói đơn giản 1 service.</p>
<h2>▸ Specialist·V1 — ③ Thành thạo</h2>
<p><strong>Khác Ex·V2:</strong> viết được Dockerfile phức tạp nhiều bước và dùng <strong>volume</strong> để giữ dữ liệu bền vững qua các lần chạy container.</p>
<p><strong>Ví dụ thực tế — mount volume để dữ liệu CSDL không mất khi container bị xoá.</strong></p>
<pre><code class="language-yaml"><span class="hljs-attr">services:</span>
  <span class="hljs-attr">db:</span>
    <span class="hljs-attr">image:</span> <span class="hljs-string">postgres:16</span>
    <span class="hljs-attr">volumes:</span>
      <span class="hljs-bullet">-</span> <span class="hljs-string">pgdata:/var/lib/postgresql/data</span>   <span class="hljs-comment"># dữ liệu lưu NGOÀI container, trên máy host</span>
<span class="hljs-attr">volumes:</span>
  <span class="hljs-attr">pgdata:</span>
</code></pre>
<p>Không có volume, dữ liệu CSDL nằm trong lớp ghi (writable layer) của container — <code>docker compose down</code> xoá container là mất sạch dữ liệu. Với volume, dữ liệu tồn tại độc lập với vòng đời container, container chết/tạo lại vẫn còn nguyên dữ liệu cũ.</p>
<p><strong>Vì sao là mức ③:</strong> bạn viết được Dockerfile/compose phức tạp và hiểu đúng cách giữ dữ liệu bền vững — không chỉ đóng gói ứng dụng không trạng thái.</p>
