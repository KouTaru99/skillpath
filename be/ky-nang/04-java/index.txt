18:T17b6,<h1>Java (ngôn ngữ lập trình)</h1>
<p><strong>Định nghĩa.</strong> Java là ngôn ngữ hướng đối tượng chạy trên <strong>JVM</strong> (Java Virtual Machine — máy ảo giúp cùng một chương trình chạy được trên nhiều hệ điều hành), phổ biến nhất cho Back-end doanh nghiệp tại Việt Nam (bao gồm Viettel/VCS), thường đi kèm framework <strong>Spring Boot</strong>. Trang này minh hoạ bằng Java/Spring Boot xuyên suốt — dù career-path liệt kê thêm C/C++, Python, Go là các lựa chọn khác.</p>
<h2>▸ Entry — ② Biết làm</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Viết code Java rõ ràng, nắm nguyên tắc <strong>SOLID</strong> cơ bản, xử lý ngoại lệ (exception) đúng cách, tuân coding convention.</p>
<p><strong>Ví dụ thực tế — xử lý ngoại lệ đúng cách khi tìm đơn hàng không tồn tại.</strong></p>
<pre><code class="language-java"><span class="hljs-meta">@GetMapping(&quot;/api/orders/{id}&quot;)</span>
<span class="hljs-keyword">public</span> Order <span class="hljs-title function_">getOrder</span><span class="hljs-params">(<span class="hljs-meta">@PathVariable</span> Long id)</span> {
  <span class="hljs-keyword">return</span> orderRepository.findById(id)
      .orElseThrow(() -&gt; <span class="hljs-keyword">new</span> <span class="hljs-title class_">OrderNotFoundException</span>(<span class="hljs-string">&quot;Không tìm thấy đơn hàng: &quot;</span> + id));
}

<span class="hljs-comment">// ❌ KHÔNG nên: bắt Exception rồi nuốt lỗi, không ai biết chuyện gì xảy ra</span>
<span class="hljs-keyword">try</span> {
  <span class="hljs-keyword">return</span> orderRepository.findById(id).get();
} <span class="hljs-keyword">catch</span> (Exception e) {
  <span class="hljs-keyword">return</span> <span class="hljs-literal">null</span>;   <span class="hljs-comment">// caller nhận null, không biết là lỗi hay đơn hàng thật sự rỗng</span>
}
</code></pre>
<p><code>orElseThrow</code> với exception có tên rõ ràng (<code>OrderNotFoundException</code>) giúp lỗi được xử lý đúng chỗ (Spring tự dịch thành HTTP 404 nếu có <code>@ExceptionHandler</code>), thay vì nuốt lỗi âm thầm bằng <code>catch (Exception e)</code> chung chung.</p>
<p><strong>Vì sao là mức ②:</strong> viết code rõ ràng, xử lý ngoại lệ có chủ đích — đã đủ để làm task được giao mà không cần người khác chỉnh lại nhiều.</p>
<h2>▸ Ex·V1 — ③ Thành thạo</h2>
<p><strong>Khác Entry:</strong> thành thạo ngôn ngữ đủ để tự đọc hiểu tài liệu và code các chức năng phức tạp mà không cần hướng dẫn.</p>
<p><strong>Ví dụ thực tế — dùng Stream API để xử lý dữ liệu gọn, thay vì vòng lặp thủ công.</strong></p>
<pre><code class="language-java"><span class="hljs-comment">// ❌ Vòng lặp thủ công dài dòng</span>
List&lt;String&gt; names = <span class="hljs-keyword">new</span> <span class="hljs-title class_">ArrayList</span>&lt;&gt;();
<span class="hljs-keyword">for</span> (Order o : orders) {
  <span class="hljs-keyword">if</span> (o.getStatus().equals(<span class="hljs-string">&quot;COMPLETED&quot;</span>)) {
    names.add(o.getCustomerName());
  }
}

<span class="hljs-comment">// ✅ Stream API — ngắn gọn, rõ ý định</span>
List&lt;String&gt; names = orders.stream()
    .filter(o -&gt; o.getStatus().equals(<span class="hljs-string">&quot;COMPLETED&quot;</span>))
    .map(Order::getCustomerName)
    .collect(Collectors.toList());
</code></pre>
<p>Bạn dùng đúng tính năng ngôn ngữ (Stream, Optional, generics) để code ngắn gọn và ít lỗi hơn, không chỉ viết được mà viết đúng &quot;chất Java&quot;.</p>
<p><strong>Vì sao là mức ③:</strong> thành thạo ngôn ngữ, code các chức năng có logic phức tạp mà không cần người hướng dẫn.</p>
<h2>▸ Ex·V3 — ④ Chuyên sâu</h2>
<p><strong>Khác V1:</strong> phân tích và <strong>tối ưu hiệu năng</strong> dựa trên hiểu biết sâu về cách Java thực thi — không chỉ viết code đúng chức năng.</p>
<p><strong>Ví dụ thực tế — phát hiện và sửa vấn đề tạo object thừa trong vòng lặp nóng (hot loop).</strong></p>
<pre><code class="language-java"><span class="hljs-comment">// ❌ Tạo mới StringBuilder ở mỗi vòng lặp — tốn bộ nhớ, tăng áp lực Garbage Collector</span>
<span class="hljs-keyword">for</span> (Order o : orders) {
  <span class="hljs-type">StringBuilder</span> <span class="hljs-variable">sb</span> <span class="hljs-operator">=</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">StringBuilder</span>();   <span class="hljs-comment">// 10.000 đơn = 10.000 object bị tạo rồi bỏ</span>
  sb.append(o.getId()).append(<span class="hljs-string">&quot;:&quot;</span>).append(o.getStatus());
  results.add(sb.toString());
}

<span class="hljs-comment">// ✅ Dùng String.format hoặc text block đơn giản, hoặc tái sử dụng builder nếu vòng lặp lớn</span>
<span class="hljs-keyword">for</span> (Order o : orders) {
  results.add(o.getId() + <span class="hljs-string">&quot;:&quot;</span> + o.getStatus());   <span class="hljs-comment">// JVM tự tối ưu nối chuỗi đơn giản</span>
}
</code></pre>
<p>Với vòng lặp xử lý hàng chục nghìn bản ghi, tạo object thừa liên tục làm Garbage Collector phải dọn dẹp thường xuyên hơn, ảnh hưởng hiệu năng tổng thể — bạn nhận ra và sửa nhờ hiểu cách JVM quản lý bộ nhớ, không chỉ nhờ code chạy đúng.</p>
<p><strong>Vì sao là mức ④:</strong> bạn hiểu đủ sâu cơ chế thực thi của Java để tối ưu hiệu năng thực chất — mức cao nhất của kỹ năng ngôn ngữ trong thang Entry→Experienced.</p>
