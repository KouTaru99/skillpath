18:Ta2e,<h1>CI/CD cơ bản</h1>
<p><strong>Định nghĩa.</strong> <strong>CI/CD</strong> (Continuous Integration / Continuous Deployment — tích hợp &amp; triển khai liên tục) là quy trình tự động build, chạy test, và triển khai code mỗi khi có thay đổi, thay vì làm thủ công. Khác Dev, Tester <strong>không vận hành pipeline</strong> — chỉ cần hiểu đủ để biết bản build mình đang test tới từ đâu, đọc được log khi pipeline fail, và (mức cao hơn) gắn test tự động của mình vào pipeline. Với Tester, CI/CD xuất hiện muộn (từ Senior) đúng vì trước đó công việc chưa chạm tới nó.</p>
<h2>▸ Senior·V1 — ① Nhập môn</h2>
<p><strong>Ở mức này bạn làm chủ được gì.</strong> Hiểu các bước cơ bản của pipeline (build → test → deploy), đọc được log khi fail và định vị đúng lỗi thuộc bước nào — để báo đúng người, không đổ nhầm việc.</p>
<p><strong>Tình huống thực tế — pipeline staging fail, bạn đọc log định vị đúng chỗ.</strong> Build mới của hệ thống giám sát an ninh mạng lên staging thất bại. Thay vì báo chung chung &quot;deploy hỏng rồi&quot;, bạn mở log CI/CD đọc:</p>
<pre><code>PIPELINE #482 — FAILED
  ✓ Stage 1: Checkout code          (12s)
  ✓ Stage 2: Build                  (1m 40s)
  ✗ Stage 3: Run unit tests         (FAILED)
        FAIL  AlertCorrelationTest.shouldMergeWithin5Min
        expected 1 incident but found 2
  ⊘ Stage 4: Deploy to staging      (SKIPPED — không chạy vì stage 3 fail)

BẠN ĐỌC RA:
  - Fail ở &quot;run unit tests&quot;, KHÔNG phải build hay deploy.
  - Test fail là AlertCorrelationTest → lỗi LOGIC gộp cảnh báo,
    không phải lỗi cấu hình hạ tầng.
  → Báo cho DEV (lỗi code logic), KHÔNG báo DevOps (không phải
    lỗi deploy). Kèm luôn tên test + thông điệp fail để dev vào thẳng.
</code></pre>
<p>Giá trị của mức này: đọc đúng tín hiệu pipeline giúp gửi vấn đề tới <strong>đúng người ngay từ đầu</strong> — tránh vòng vo &quot;deploy hỏng → DevOps kiểm tra → hoá ra lỗi code → chuyển lại Dev&quot; mất nửa ngày.</p>
<p><strong>Vì sao là mức ①:</strong> hiểu đủ để đọc tín hiệu pipeline và định vị đúng vấn đề; chưa cần tự cấu hình/vận hành pipeline (việc của Dev/DevOps) — với Tester đây đã là đủ cho vai trò phối hợp.</p>
